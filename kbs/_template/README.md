# `kbs/_template/` — starter for a new KB

Copy this directory to `kbs/<new-name>/`, fill in the slots below, then run
`just kb-publish <new-name>`. See [`kbs/README.md`](../README.md) for the
full workflow.

## Slots to fill in `<new-name>/README.md`

- **Name** — slug used everywhere (must match directory)
- **Intent** — one paragraph describing what queries this KB should answer
- **Sources of authority** — primary docs, manpages, canonical repos
- **Out of scope** — what NOT to crawl
- **Refresh cadence** — when do we rebuild

## Slots to fill in `<new-name>/CLAUDE.md`

Agent-facing notes:
- What to retrieve when the KB is queried
- Known gaps or stale areas
- Preferred citation style in answers

## Required files after `just kb-publish <new-name>`

- `index.dvc` — DVC pointer for the LanceDB index dir
- `chunks.dvc` — DVC pointer for the chunks JSONL
- `.gitignore` — auto-generated by `dvc add`; commit it
- `manifest.yaml` — symlink to `~/db/kb/<new-name>/manifest.yaml`
- `README.md` — human notes (this file, customized)
- `CLAUDE.md` — agent notes
