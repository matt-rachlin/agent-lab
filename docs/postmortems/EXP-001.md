# Postmortem: EXP-001 — The 12 GB Agent v0.1

Date: 2026-05-25
Experiment: EXP-001 (`docs/exp/EXP-001.md`)
Finding: [F-003](../findings/F-003-12gb-agent-v0.1.md)

## What happened

Ran a 1,152-cell sweep across 6 models × 24 PBS-v0.1 tasks × N=8 seeds in ~4 hr 20 min. Three of four pre-registered hypotheses were refuted; the fourth (knowledge near-parity) was confirmed. No methodological failures escalated to F-003 verdicts being unsafe.

## What went well

- **Pre-registration discipline held.** Plan committed to git → registered via `lab exp register` with SHA recording before any cell ran. Hypotheses and verdict rules were not edited mid-experiment.
- **Resumability worked.** Two separate process kills (manual cleanup + qwen3 fmt failures) → re-launched with same command → matrix expansion + done-detection skipped completed cells correctly.
- **Pre-mortem caught the qwen3 reasoning issue.** Item #1 in the EXP-001 pre-mortem flagged "Qwen3 reasoning ate the 1024 token budget". That's exactly what bit H3 — and the mitigation ("report not_empty rate per model in F-003") was followed without scrambling.

## What went wrong

### 1. Stale GPU lease blocked resume after process kill (medium)

After Ctrl-C'ing the first sweep attempt to fix VRAM contention, the Valkey GPU-lease key (`lab:gpu:lease`) didn't get released — the lease has a 30-min TTL, so the second sweep blocked waiting for it. Manually `DEL`'d the key to unblock.

**Root cause:** `gpu_lease` is a `@contextmanager` that releases on context exit. On SIGTERM/SIGKILL the cleanup doesn't run.

**Fix queued for next phase:** wrap the sweep main with signal handlers that release the lease on SIGTERM. Also add `lab sweep cancel` CLI that DELs the lease as a forcing function.

### 2. LiteLLM defaulted to `keep_alive=Forever`, breaking model swaps (high)

When the sweep moved between models, Ollama tried to load model B alongside model A still hot, OOMing the 12 GB card. All 24 errors traced to this. Adding `keep_alive: 5m` per local model in the LiteLLM config solved it cleanly.

**Root cause:** newer Ollama versions appear to keep models indefinitely if no explicit keep-alive is sent. Earlier sessions hit this in interactive use without me noticing.

**Fix applied:** `conf/litellm-config.yaml` now has `keep_alive: 5m` on every local-Ollama-backed model. Committed.

**Followup todo:** add a startup check to `lab sweep run` that fails fast if the proxy config doesn't include `keep_alive`. (PHASES_TODO cross-cutting bucket.)

### 3. The 24 errors clustered, weren't randomly distributed (informational)

All 24 errors were qwen3-14b-q4 × fmt-001/002/003 × 8 seeds — the *first three tasks alphabetically* for qwen3, which is the *last model alphabetically*. Once phi4's VRAM fully released, qwen3 loaded and ran cleanly through fmt-004 onward.

This is a feature of the sweep's outer-loop-by-model ordering — it concentrates failures, which is good for diagnosis. A random ordering would have scattered the same 24 failures across the matrix and made the root cause harder to spot. Keep model-outer.

### 4. The sweep config doesn't capture "system prompt", so qwen3 ran with reasoning default-on (high — but pre-meditated)

The pre-mortem named this. It still cost H3 its predicted result. The fix is EXP-001b, deliberately variation-isolating reasoning-mode and token-budget.

### 5. Empty responses pass `latency_under` but not `not_empty` (minor)

A reasoning-blown-budget call still returns 200 OK with empty content — fast latency, zero output. The `latency_under` eval pass'd these cells but they're useless. Doesn't change verdicts (we use exact_match/regex_match), but it does mean the latency dashboard can give a falsely-rosy view of qwen3. **Followup:** add a compound evaluator `useful_latency = latency_under AND not_empty`.

## What I'd do differently

- **Run a 10-cell smoke matrix before kicking off the 1152-cell sweep** to surface VRAM contention while the model-swap interaction is still observable. The full 4-hour sweep amortizes the diagnosis cost, but a 10-min smoke would have caught keep_alive=Forever in 60 seconds.
- **Set `max_tokens=2048` by default** for sweeps that include reasoning-by-default models. Pay the extra wall time; remove the budget-pressure confound.
- **Time my "verbal estimate" against actual.** I said 7 hr → it took 4 hr 20 min. I overestimated, probably because the cloud calls were faster than I expected (Ollama Cloud Pro tier was responsive). Calibrate.

## Numbers the next sweep should target

- N=16 (not N=8) — to give Welch's t-test enough power for an effect like H1's +10pp delta
- 30+ tasks — adding 6 to PBS-v0.1 to hit the 40-60% pass@1 difficulty band that surfaces F-002's reliability collapse
- `max_tokens=2048` — to remove the budget-pressure confound on reasoning models
- Two prompt variations — neutral vs `/no_think` for qwen3-class models
